# План: Компиляция и запуск Android-приложения FamilyGuard

## Текущее состояние
- Android-проект находится в `android/`, написан на Kotlin + Jetpack Compose + Hilt
- Gradle-проект **неинициализирован**: нет `settings.gradle.kts`, `gradlew`, `gradle.properties`, каталога версий (`libs.versions.toml`)
- Нет готового APK/AAB
- Backend и frontend готовы к запуску через Docker Compose и `npm run dev`

## Способы запуска

### Вариант A: Через Android Studio (рекомендуется)
1. Установить Android Studio (Giraffe+), при установке выбрать Android SDK API 35, NDK, Kotlin
2. Открыть папку `android/` как проект — Android Studio сама сгенерирует недостающие Gradle-файлы
3. Дождаться полной синхронизации Gradle (скачает зависимости)
4. В `app/src/main/java/com/familyguard/app/data/local/TokenStorage.kt` и других местах проверить, что нет ошибок
5. Собрать APK: **Build → Build Bundle(s)/APK(s) → Build APK(s)** или **Run → Run 'app'** на эмуляторе/устройстве
6. Для отладки используйте эмулятор Pixel с API 34+ (minSdk 26, compileSdk 35)

### Вариант B: Через командную строку (после инициализации Gradle)
1. Сгенерировать `gradlew` и обёртку:
   ```bash
   cd android
   gradle wrapper --gradle-version 8.5
   ```
2. Создать `settings.gradle.kts` и `libs.versions.toml` на основе зависимостей из `app/build.gradle.kts`
3. Собрать debug APK:
   ```bash
   ./gradlew assembleDebug
   ```
4. Установить на подключённое устройство:
   ```bash
   ./gradlew installDebug
   ```

### Вариант C: Запуск без компиляции (как веб-приложение)
Поскольку Android-клиент — это полноценное нативное приложение (Compose, VPN, Device Admin, Accessibility Service), **полноценно запустить его без компиляции невозможно** в браузере или через Expo.  
Однако можно запустить только **backend + frontend** и тестировать API/дашборд родителя:
```bash
# 1. Запустить backend
docker compose up -d
docker compose exec web python manage.py createsuperuser

# 2. Запустить frontend (дашборд)
cd frontend && npm install && npm run dev
# → http://localhost:3000
```

## Критичные замечания
- Для VPN и Accessibility Service требуется физическое Android-устройство (minSdk 26, Android 8+), эмулятор не поддерживает VPN-tunnel и Device Admin в полной мере
- `app/build.gradle.kts:26` уже настроен `10.0.2.2:8000` для debug-сборки (эмулятор), для физического устройства нужно заменить на IP хоста
- Не хватает keystore для release-подписи
